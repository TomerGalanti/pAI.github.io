Distribution-aware synthesis

pAI replaces the marked region with a solver specialized to the benchmark distribution.

The application keeps the same interface. pAI observes repeated instances, synthesizes candidate implementations, validates them, and deploys the selected solver only after it passes feasibility and performance checks.

def solve(graph):
    # Public contract: return a valid dominating set.
    n = graph.num_vertices()
    deg = graph.degrees()
    # pAI-managed region
    uncovered = all_vertices(graph)
    solution = []

    while uncovered:
        v = best_greedy_vertex(graph, uncovered)
        solution.append(v)
        uncovered -= graph.closed_neighbors(v)

    solution = repair_and_prune(graph, solution)
    # end pAI-managed region
    assert is_dominating_set(graph, solution)
    return solution
def solve(graph):
    # Public contract: return a valid dominating set.
    n = graph.num_vertices()
    deg = graph.degrees()
    # pAI-managed region (synthesized)
    core = degree_bucket_seed(graph, deg)
    solution = distribution_tuned_greedy(
        graph,
        seed=core,
        lookahead=2,
        prune_budget=1400,
    )

    solution = private_pace_repair(graph, solution)
    solution = local_delete_prune(graph, solution)
    # end pAI-managed region
    assert is_dominating_set(graph, solution)
    return solution

Takeaways

The result is strongest as a speed–quality tradeoff.

External validation

The solver is evaluated on PACE 2025 private instances — a serious external benchmark rather than a toy distribution.

Correct claim

The method does not beat the released PACE solvers on pure size. It gets close while running about two orders of magnitude faster.

Why synthesis matters

A best@5 Codex baseline is valid but slower and returns larger dominating sets. The evaluation loop matters.
